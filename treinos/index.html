1
⚙️ Instalando as ferramentas
5 min
1.1 Abrir o Terminal

No seu Mac:

  • Aperta Cmd + Espaço (abre o Spotlight)
  • Digita Terminal
  • Aperta Enter

Vai abrir uma janela preta ou cinza. Não tem segredo: você vai colar uns comandos aqui.

1.2 Instalar o Homebrew

Cola esse comando no Terminal e aperta Enter:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

O que vai acontecer:

  • Vai pedir sua senha do Mac (a mesma que você usa pra fazer login)
  • Digita ela e aperta Enter — não aparece nada na tela enquanto digita, é normal!
  • Espera terminar (3–5 minutos)

Quando terminar, vai aparecer uma mensagem como esta:

==> Next steps: - Run these commands in your terminal to add Homebrew to your PATH: echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/SEUNOME/.zprofile eval "$(/opt/homebrew/bin/brew shellenv)"
1.3 Adicionar Homebrew ao sistema

Cola esses dois comandos, um de cada vez:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
source ~/.zprofile

Testa se funcionou:

brew --version
✅Se aparecer algo como Homebrew 4.x.x, tá certo!
1.4 Instalar o Python 3.11
brew install python@3.11
☕Espera terminar (pode demorar 5 minutos). Aproveita pra fazer um café.
2
🐍 Instalando o conector Garmin
5 min
2.1 Baixar o programa

Cola esses comandos um de cada vez:

mkdir -p ~/garmin_mcp_server
cd ~/garmin_mcp_server
git clone https://github.com/Taxuspt/garmin_mcp.git .
2.2 Criar ambiente Python isolado
/opt/homebrew/bin/python3.11 -m venv venv
source venv/bin/activate
ℹ️Você vai notar que o terminal agora começa com (venv) — isso é normal e indica que tá tudo certo.
2.3 Instalar dependências
pip install --upgrade pip
pip install -e .

Espera terminar.

✅Quando aparecer Successfully installed garmin-mcp…, tá pronto!
3
🔐 Conectando com sua conta Garmin
2 min
3.1 Autenticar

Cola esse comando:

garmin-mcp-auth

O que vai acontecer:

  1. Vai pedir seu email do Garmin Connect
  2. Vai pedir sua senha do Garmin
  3. Se você tem autenticação de 2 fatores (MFA) ativada no Garmin, vai pedir o código

Quando terminar com sucesso, você vai ver:

✓ Authentication successful! Logged in as: SEU NOME SUCCESS: You can now use the Garmin MCP server!
🎉Os tokens de acesso foram salvos no seu Mac e duram 6 meses.
4
⚙️ Configurando o Claude Desktop
3 min
4.1 Descobrir seu nome de usuário
whoami

Vai aparecer seu nome de usuário do Mac (exemplo: joaopaulo). Anota porque você vai usar ele agora.

4.2 Abrir o arquivo de configuração
open ~/Library/Application\ Support/Claude/claude_desktop_config.json

Vai abrir um editor de texto com um arquivo JSON.

4.3 Substituir o conteúdo

APAGA TUDO que tá no arquivo e cola exatamente isso:

{ "mcpServers": { "garmin-local": { "command": "/Users/SEUNOME/garmin_mcp_server/venv/bin/garmin-mcp", "env": { "GARMIN_COOKIES_FILE": "/Users/SEUNOME/.garminconnect/cookies.json" } } } }
⚠️
IMPORTANTE: Substitua SEUNOME (em 2 lugares) pelo nome que apareceu no comando whoami.

Exemplo: se seu usuário é joaopaulo, troca SEUNOME por joaopaulo nos dois lugares.
4.4 Salvar

Aperta Cmd + S para salvar.

4.5 Reiniciar Claude
  1. Feche o Claude Desktop completamente (Cmd + Q)
  2. Abra de novo

Solução de problemas
"Server disconnected" no Claude ▼

1. Verifica se você substituiu SEUNOME pelo seu usuário em AMBOS os lugares no arquivo de config.

2. Confirma que rodou o garmin-mcp-auth com sucesso (Parte 3).

3. No Claude Desktop, vai em Configurações → Servidores MCP locais e clica em "Ver Logs" no servidor garmin-local. Lê a mensagem de erro.

"Command not found: brew" ▼

Você pulou o Passo 1.3. Roda esses dois comandos:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
source ~/.zprofile
"Module not found: garmin_mcp" ▼

Provavelmente o ambiente Python não foi ativado. No Terminal:

cd ~/garmin_mcp_server source venv/bin/activate pip install -e .
Tokens expiraram (depois de ~6 meses) ▼

Roda de novo:

cd ~/garmin_mcp_server source venv/bin/activate garmin-mcp-auth
Outro erro? Dica de ouro ▼

O Claude pode te ajudar a resolver erros de instalação dele mesmo. Funciona muito bem!

1. No Claude Desktop, abre uma nova conversa (fora do projeto Treinador)

2. Cola algo assim:

Tô seguindo um tutorial pra conectar meu Garmin ao Claude Desktop via MCP no Mac. Tô travado no passo X e aparece esse erro: [COLA O ERRO AQUI]. Me ajuda a resolver passo a passo, considerando que sou leigo em tecnologia.

3. Se o erro for em algum log do Claude Desktop, vai em Configurações → Servidores MCP locais → Ver Logs, copia tudo e cola na conversa.

💡O Claude consegue diagnosticar a maioria dos problemas e te dar o comando exato pra resolver.

Bônus — Projeto "Treinador"

Pra deixar o Claude ainda mais útil, configura instruções personalizadas no seu projeto.
Vai em Projeto → Instruções e cola o texto abaixo (editando seus dados entre colchetes):

Você é meu treinador pessoal de corrida. CONTEXTO SOBRE MIM: - Idade: [SUA IDADE] - Sexo: [M/F] - Peso: [SEU PESO] - Anos correndo: [QUANTOS ANOS] - Objetivo principal: [EX: maratona em outubro] - Frequência semanal de treinos: [EX: 4x na semana] COMO QUERO QUE VOCÊ ME AJUDE: - Use meus dados reais do Garmin (via MCP) sempre que possível - Foque em análises práticas, não teoria genérica - Aponte sinais de overtraining ou subtreino - Sugira ajustes específicos com base no que vê nos meus dados - Seja direto, sem rodeios EVITE: - Conselhos genéricos do tipo "treine com consistência" - Recomendações sem considerar meus dados - Linguagem motivacional vazia

🔒 É seguro?

Roda localmente

Tudo no seu Mac — nenhum servidor de terceiros acessa seus dados.

Tokens seguros

Credenciais ficam em ~/.garminconnect só no seu computador.

Dados da fonte oficial

Lidos direto do servidor do Garmin Connect.

Como desconectar

Delete ~/garmin_mcp_server e ~/.garminconnect.
